# 🏨 酒店集团舆情情报综合工具箱

> 通用版 | v3.1 | 支持任意酒店集团复用
> 适配：华住会/锦江/亚朵/首旅如家/德胧/开元/及其他酒店集团
> 无硬编码 | 可配置 | 可开源

## ⚙️ 配置（使用前必读）

### 品牌配置化

所有品牌相关配置通过 `BRAND_CONFIG` 变量传入，**无需修改代码**：

```javascript
// 德胧版示例
{
  brand: "德胧集团",
  brands: ["开元名都", "钛唐", "开元酒店"],
  competitors: ["华住会", "锦江集团", "亚朵", "首旅如家"],
  feishu_user_id: "ou_xxxxxxxx",  // 日报推送目标
  // 两层架构配置
  risk: { platforms: [...], schedule: "0 */2 * * *" },
  ota: { platforms: [...], schedule: "0 10 * * *", bitable_id: "xxx" }
}
```

### 默认配置 vs 可调整

| 配置项 | 默认值 | 可调整 |
|--------|--------|--------|
| 舆情扫描频率 | 每2小时 | 可改为每1/4/6小时 |
| 差评采集频率 | 每日10点 | 可改为任意时间 |
| 监控平台 | 黑猫/微博/抖音 | 可增删平台 |
| 竞品列表 | 华住/锦江/亚朵/首旅如家 | 可增删竞品 |
| 推送目标 | 指定用户 | 可改为群聊 |
| 严重度阈值 | >6入库 | 可调整 |
| 差评维度 | 12个 | 可增删类型 |

### 交互调整示例

**随时告诉龙虾你想要什么调整**：

```
「调低舆情扫描频率」→「改为每4小时一次」
「增加竞品监控」→「加入锦江AI动态」
「切换推送目标」→「改为推送到群 oc_xxx」
「暂停舆情扫描」→「暂时停止定时任务」
「增加视频平台」→「加入B站和快手」
「调整差评阈值」→「严重度>8才入库」
「恢复默认配置」→「重新启用原始Cron」
```

## 🏗️ 系统架构（两层分离）

```
酒店舆情情报综合工具箱
│
├── 数据源层
│   ├── miaoda-web-search（主力搜索）
│   ├── BettaFish 扩展（深度舆情）
│   └── 视频平台专项（抖音/快手/B站/小红书）
│
├── 分析层
│   ├── 舆情分类（投诉/安全/服务/运营/AI）
│   ├── 风险评级（🔴P0/🟠P1/🟡P2/🔵P3）
│   ├── 5Why根因分析
│   └── 防控预案生成
│
├── 【舆情风险层】← 快捷扫描
│   ├── 定位：公开平台紧急舆情
│   ├── 平台：黑猫/微博/抖音/知乎/新闻
│   ├── Cron：每2小时（可调）
│   └── 输出：飞书预警卡片 → 私信
│
└── 【差评维度层】← 分类管理
    ├── 定位：OTA渠道内部差评
    ├── 平台：携程/美团/飞猪/小红书
    ├── Cron：每日10点（可调）
    ├── 分类：12个差评维度
    └── 输出：飞书多维表格 → 周报统计
```

## 🚀 快速使用

### 舆情采集（传入品牌配置）

```bash

  --instruction "提取投诉内容、媒体来源、传播范围、情感倾向"
```

### 竞品动态采集

```bash
miaoda-studio-cli search-summary --query "华住 AI动态定价 2026" \
  --instruction "提取AI覆盖数据、效果指标、竞品对比"
```

## 🛠️ 工具四：AI风险分析与防控预案

### 分析流程

```
舆情数据 → 舆情分类 → 风险评级 → 5Why分析 → 防控预案 → 飞书卡片
```

### 通用风险分析Prompt

````
请分析以下舆情事件，生成风险报告：

## 舆情内容
{粘贴舆情内容}

## 品牌信息
- 品牌：{品牌名}
- 涉事门店：{门店名}
- 事件类型：{类型}

## 分析要求
1. 风险评级（🔴高危/🟠中危/🟡低危）
2. 风险归因（5Why法）
3. 防控预案：
   - 预防措施（具体可执行）
   - 监控指标（可量化阈值）
   - 应急预案（触发条件+处置流程）
4. 同类舆情规避建议
````

## 📊 差评维度分类体系（12维度）

| 维度 | 图标 | 严重度 | 说明 |
|------|------|--------|------|
| 设施老旧 | 🏚️ | 6 | 硬件陈旧、维修不到位 |
| 卫生问题 | 🛁 | 8 | 清洁不彻底、异味、虫害 |
| 噪音问题 | 🔊 | 5 | 隔音差、噪音扰民 |
| 前台态度 | 😤 | 7 | 服务态度恶劣、冷暴力 |
| 服务效率 | 🐌 | 6 | 入住慢、响应迟缓 |
| 虚假宣传 | 🎭 | 9 | 货不对板、夸大宣传 |
| 积分克扣 | 💰 | 8 | 积分异常扣减 |
| 权益缩水 | 📉 | 7 | 会员权益降级 |
| 退款纠纷 | 💸 | 7 | 取消扣款、退款拖延 |
| 临时涨价 | 📈 | 8 | 节假日涨价、临时加价 |
| 隐私泄露 | 🔓 | 10 | 信息泄露、隐私侵犯 |
| 安全隐患 | ⚠️ | 10 | 消防/安全/事故隐患 |

## ⏰ 自动化配置

### 舆情风险层Cron（快捷扫描）

```javascript
{
  name: "{品牌}舆情风险快捷扫描",
  schedule: {kind: "cron", expr: "0 */2 * * *", tz: "Asia/Shanghai"},
  sessionTarget: "isolated",
  payload: {
    kind: "agentTurn",
    message: "执行{品牌}舆情风险快捷扫描：\n1. miaoda搜索黑猫投诉最新\n2. 微博热搜相关\n3. 抖音/快手视频舆情\n4. 检查新增P0/P1级舆情\n5. 如有新舆情：写入飞书多维表格【舆情风险层】+ 生成预警卡片推送\n6. 如无新舆情，输出「本次扫描无新增」"
  }
}
```

### 差评维度层Cron（分类管理）

```javascript
{
  name: "{品牌}OTA差评维度采集",
  schedule: {kind: "cron", expr: "0 10 * * *", tz: "Asia/Shanghai"},
  sessionTarget: "isolated",
  payload: {
    kind: "agentTurn",
    message: "执行{品牌}OTA差评维度采集：\n1. miaoda搜索携程/美团/飞猪最新评价\n2. 小红书素人笔记\n3. 自动分类到12个差评维度\n4. 写入飞书多维表格【差评维度层】\n5. 生成统计卡片推送"
  }
}
```

### 交互调整关键词

| 用户说 | 龙虾会做什么 |
|--------|-------------|
| 调低扫描频率 | 修改舆情Cron为更长间隔 |
| 增加竞品 | 在竞品列表加入新品牌 |
| 切换推送目标 | 修改delivery的to为新用户/群 |
| 暂停任务 | 禁用对应Cron |
| 增加平台 | 在platforms列表加入新平台 |
| 调整阈值 | 修改严重度阈值 |
| 恢复默认 | 重新启用原始配置 |

## 📦 飞书多维表格结构

### 舆情风险层（tblxxx）

| 字段 | 类型 | 说明 |
|------|------|------|
| 风险标题 | 文本 | 事件简述 |
| 事件摘要 | 文本 | 详细描述 |
| 风险级别 | 单选 | 🔴P0/🟠P1/🟡P2/🔵P3 |
| 风险类型 | 多选 | 投诉/安全/服务/卫生等 |
| 涉及品牌 | 单选 | 德胧/华住/锦江等 |
| 涉及酒店 | 文本 | 具体门店 |
| 发生平台 | 多选 | 黑猫/微博/抖音等 |
| 原文链接 | 超链接 | 原始链接 |
| 发现时间 | 日期 | 抓取时间 |
| 处理状态 | 单选 | 待处理/处理中/已处置 |

### 差评维度层（tblxxx）

| 字段 | 类型 | 说明 |
|------|------|------|
| 差评来源 | 文本 | 内容摘要 |
| 差评类型 | 多选 | 12维度 |
| 严重度 | 数字 | 1-10 |
| 涉及品牌 | 单选 | 品牌归属 |
| 涉及酒店 | 文本 | 具体门店 |
| OTA渠道 | 单选 | 携程/美团等 |
| 入住日期 | 日期 | 入住时间 |
| 差评时间 | 日期 | 发布时间 |
| 整改进度 | 单选 | 待处理/已整改 |
